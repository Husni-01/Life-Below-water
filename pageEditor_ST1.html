Splash Page

Technical Description

Implementation of a Splash Screen Using HTML and CSS

For this project, I developed a splash screen that effectively introduces the theme of "Life Below Water." The primary objective was to create a visually engaging experience by using structured HTML elements and CSS animations. By incorporating a background video, animated logo elements, a loading effect, and a well-organized layout, I ensured that the splash screen captured user attention while maintaining clarity and usability.

1. Key HTML Tags for Structure

The foundation of the splash screen is built using various HTML tags that structure the content logically and efficiently.

a. <video> for Background Animation
To create an immersive experience, I included a background video that continuously loops behind the content. The <video> element was used for this, allowing the integration of a moving background while keeping it muted and autoplaying to avoid user disruption.

b. <div> Containers for Layout Organization
Several <div> elements were used to structure different sections of the splash screen. These include:
- A logo container, which holds individual parts of the animated logo.
- A loader container, which houses the loading bar animation.
- A background overlay, which helps enhance readability by adding a subtle dark layer over the background video.

c. <h1>, <h2>, and <h3> for Text Content
To create a well-structured heading hierarchy:
- <h1> was used for the main title, making it the most prominent text on the page.
- <h2> was used for the mission statement, ensuring a slightly smaller but still highly visible text block.
- <h3> was used for subtitles, such as the "Our Members" section.

d. <ul> and <li> for Member List
To present the names of the team members in a structured way, I used an unordered list (<ul>) with list items (<li>). Instead of displaying them in a traditional vertical format, I styled them using CSS to appear in a horizontal layout.

2. Key CSS Properties for Styling and Animation

The visual appeal of the splash screen was achieved through CSS properties that controlled layout, positioning, color schemes, and animations.

a. Background and Overlay Styling
To ensure readability over the background video, I applied a semi-transparent overlay:
- background-color: rgba(0, 0, 0, 0.3); – This adds a subtle dark tint without completely blocking the video.
- position: absolute; top: 0; left: 0; width: 100%; height: 100%; – Ensures the overlay covers the entire screen.

b. Logo and Image Positioning
The logo is broken into separate parts (body, tail, and waves), and each is positioned precisely using:
- position: absolute; – Ensures elements can be freely positioned relative to their parent container.
- left: 50%; transform: translate(-50%, -50%); – Centers the elements horizontally and vertically.

c. Text Formatting
To maintain a clean and modern look:
- Font styling: I used font-family: 'Oswald', sans-serif; to apply a professional, bold font.
- Text alignment: text-align: center; ensures that headings and descriptions are centered properly.
- Color contrast: color: #fff; makes the text stand out against the darker background.

d. Animations for Smooth Transitions
To create engaging effects, I applied CSS keyframe animations, such as:
- Fade-in and slide effects – Used for text and logo elements to make them appear smoothly:
  opacity: 0; transform: translateY(50px); at the start.
  opacity: 1; transform: translateY(0); at the end.
- Loading bar animation – The loading bar gradually fills up:
  animation: l6 5s forwards; where @keyframes l6 increases the width over time.
- Logo movement – Different parts of the logo enter from various directions:
  animation: slideFromLeft 0.5s ease-in-out 1s forwards; – Moves elements from the left.
  animation: slideFromTop 0.5s ease-in-out 0.3s forwards; – Moves elements from the top.

e. Responsive Design
To ensure proper display on different screen sizes:
- max-width: 100%; height: auto; – Ensures images and elements resize dynamically.
- @media screen and (max-width: 768px) – Adjusts layout for smaller screens by modifying text sizes and element positions.

Accessibility

For accessibility in this splash screen, I ensured text readability by using high contrast between the text and background. I also utilized semantic HTML tags, like <h1>, <h2>, and <ul>, to improve the page structure and make it easier for screen readers to interpret. Additionally, the background video was muted and set to autoplay to avoid unexpected sound, while the <video> tag ensures that it is still accessible to users who may prefer not to interact with media. I also made the layout responsive, adjusting elements to fit different screen sizes, improving usability for mobile users.

Volunteer Page

Technical Description

The HTML structure provides the foundation for the layout and content of the page. The page utilizes several key HTML elements that contribute to the organization and interactivity.

<header>: The <header> element is used to define the top section of the page, which typically contains the main title or navigation. It helps to provide context for users and search engines.

<section>: The <section> tag is used to group related content together. For example, the carousel, review containers, and other parts of the page are separated into sections for better organization and clarity.

<ul> and <li>: These elements are crucial for the layout of the carousel. An unordered list (<ul>) wraps the list items (<li>), each containing an image. This structure allows for easy horizontal scrolling via CSS and JavaScript.

<div>: The <div> tag is used extensively to create containers that group content and styling. For example, the .card class groups individual items such as images and text content. It helps in maintaining flexible layouts.

<button>: Buttons are defined using the <button> element, which makes them semantically correct for interactivity. The button elements are used to create clickable actions like “Learn More,” “View,” or “Apply.”

CSS is used to control the appearance of the HTML structure, and various properties contribute to both the layout and visual design of the page.

Flexbox and Grid: Flexbox (<display: flex>) is used to arrange items in a row or column, providing flexibility for layouts. In the carousel, Flexbox is used to align the images horizontally within the container. Additionally, CSS Grid is used to arrange the cards in a responsive manner. This allows content to adapt based on screen size, ensuring a consistent layout across devices.

Positioning: The position property plays a significant role in layout and element placement. position: relative is used to define an element’s position relative to its normal position, while position: absolute is used to place elements relative to their nearest positioned ancestor. This is seen in elements like the review containers and buttons, where precise placement is required.

Animation: Keyframe animations (@keyframes) are used to create smooth transitions for certain elements on the page. For example, the carousel images slide from left to right, and the images grow in size with a scaling effect. Animations like slideFromLeft and grow contribute to the dynamic nature of the page, enhancing user engagement.

Typography: Typography properties, such as font-size, font-weight, and text-align, are used to control the appearance of text. These properties ensure that headings and body text are readable and visually distinct. Larger fonts are typically used for important headings, and the sans-serif font family is applied for a modern, clean appearance.

Background and Color: The background-color and background-image properties control the visual appearance of elements' backgrounds. For instance, the body has a background image of water (url(Images/water.png)) and a background color set to a light blue (#00ccff) to match the theme of the page. Colors are also used for buttons, text, and other elements to enhance visual hierarchy and interactivity.

Borders and Shadows: The border-radius property is used to give rounded corners to various elements, such as cards and buttons, creating a modern and friendly design. The box-shadow property is applied to elements like the review containers and cards to add depth and visual separation between components.

Hover Effects: CSS hover effects are applied to buttons and interactive elements. For instance, the .learn-more:hover and .view:hover classes change the background color and scale the button when hovered, providing immediate visual feedback to users and improving interactivity.

The combination of these HTML tags and CSS properties creates a functional and aesthetically pleasing design that is both responsive and engaging for users. The layout is intuitive, and the styling choices enhance user experience by providing smooth animations, interactive buttons, and a visually appealing structure.

Accessibility

In this project, I addressed accessibility by focusing on the design elements that make the page more usable for everyone. I ensured that interactive elements like buttons and links are clearly defined, easy to focus on, and accessible via keyboard navigation. For instance, I used proper semantic HTML tags such as <button> for clickable actions, making them recognizable for assistive technologies. I also applied focus styles to interactive elements, allowing users to navigate seamlessly using keyboard or screen readers. Additionally, I ensured good contrast between text and background, helping users with visual impairments. The layout is also responsive, ensuring that the page adapts well to various devices, which is important for accessibility.

Content Page

Technical Description

HTML Structure
I used the HTML structure to create the foundation for the layout and content of the page. Several key HTML elements were chosen to help organize and make the page interactive.

- <header>: I used the <header> element to define the top section of the page, which typically contains the main title or navigation. This helps provide context for users and search engines.

- <section>: The <section> tag was used to group related content together. For example, I used it to separate parts like the carousel, review containers, and other sections for better organization and clarity.

- <ul> and <li>: These elements are crucial for the carousel layout. I wrapped the list items (<li>) inside an unordered list (<ul>), and each item contains an image. This structure makes it easy to apply horizontal scrolling with CSS and JavaScript.

- <div>: The <div> tag was used extensively to group content and styling. For instance, I used the .card class to group images and text content, which helps in creating flexible layouts.

- <button>: I used the <button> element to define buttons for interactivity. These are used for actions like “Learn More,” “View,” or “Apply.”

CSS Styling Techniques
I used CSS to control the appearance of the HTML structure. Various properties contribute to both the layout and visual design of the page.

- Flexbox and Grid: Flexbox (display: flex) was used to arrange items in a row or column, allowing flexibility in layouts. In the carousel, I used Flexbox to align images horizontally within the container. Additionally, I used CSS Grid to arrange cards responsively, adapting to different screen sizes.

- Positioning: The position property plays a significant role in element placement. For example, I used position: relative to position elements relative to their normal position, and position: absolute to place elements in relation to their nearest positioned ancestor. This was especially useful for buttons and review containers.

- Background Video: I included a background video to visually convey the message of plastic waste in the ocean. The video element with the autoplay, muted, and loop attributes ensures a continuous loop of the video without sound. The object-fit: cover property ensures the video covers the screen, providing a dynamic backdrop.

- Text Styling: I used typography properties such as font-size, font-weight, and text-align to make the headings and body text readable and visually distinct. Larger fonts were used for headings, and the sans-serif font family was applied for a clean, modern appearance.

- Background Color and Overlays: The background-color and background-image properties control the appearance of the background. The body has a light blue color and a water-themed background image, matching the page's message. I applied a dark overlay to the video to enhance text readability and make the content stand out.

- Hover Effects: I added CSS hover effects to interactive elements like buttons. For example, I applied the .learn-more:hover and .view:hover classes, changing the background color and scaling the buttons when hovered, providing visual feedback and improving interactivity.

- Animation: I used keyframe animations (@keyframes) to create smooth transitions for certain elements on the page. For instance, the carousel images slide from left to right, and I applied scaling effects to enhance user engagement.

By combining these HTML tags and CSS properties, I created a functional, interactive, and visually appealing design. The background video of plastic waste complements the messaging, while the layout and styling choices ensure a smooth user experience across devices.

Accessibility

Accessibility
I prioritized accessibility to ensure that all users, including those with disabilities, can interact with the page effectively. I implemented several techniques to improve accessibility.

- Alt Text for Images: I added descriptive alt text to all images. For instance, for the image of plastic waste in the ocean, I used <img src="Images/garbage.jpeg" alt="Garbage in the ocean">. This allows screen readers to describe the images to users with visual impairments.

- Semantic HTML: I used semantic HTML tags such as <header>, <section>, and <article> to give structure and meaning to the content. This helps screen readers and search engines understand the page layout and content, making it easier for users to navigate and access the information.

- Color Contrast: I ensured that the text color has sufficient contrast with the background for users with low vision or color blindness. For example, the white text on a dark background in the content sections helps improve readability.

- Keyboard Navigation: I made sure all interactive elements, such as buttons, can be accessed using the keyboard. This is important for users who cannot use a mouse. The <button> elements are fully navigable using the Tab key and include clear focus states for better navigation.

- Video Accessibility: For the background video, I set it to autoplay, muted, and looped to prevent unwanted sounds. Although the video is visually engaging, I ensured that it doesn’t distract or cause discomfort for users with hearing impairments.

Through these methods, I aimed to ensure the page is more accessible for a broader range of users, improving their experience while interacting with the content.

Challenges and Lessons Learned

Challenges Encountered
During the implementation of the three HTML pages, I faced several challenges that required problem-solving and a deeper understanding of both HTML and CSS. Some of the key challenges included:

- Video Background Issues: One of the primary challenges was ensuring that the background video worked seamlessly across different devices and screen sizes. At first, the video didn’t properly cover the full screen on all devices. I resolved this by using the CSS property object-fit: cover;, which ensures that the video covers the screen without distorting or leaving empty spaces, no matter the screen size.

- Responsive Design: Making sure the layout and elements adjusted correctly on all screen sizes was a challenge. Since the project used Flexbox and CSS Grid, ensuring that images, text, and sections resized correctly without breaking the design was tricky. I solved this by using relative units like percentages and viewport widths, as well as implementing media queries to fine-tune the layout for smaller screens. This allowed the design to remain functional and visually appealing across different devices.

- Overlapping Elements: In some areas, especially with the background video and content containers, elements were overlapping or being hidden behind each other. This happened because of the layering and positioning of different sections. I fixed this by carefully managing the z-index property and using position: relative; and position: absolute; to control the stacking order of elements.

- Text Visibility: Ensuring text readability over the background video was challenging, especially with the video’s dynamic nature. The text would sometimes be hard to read due to the movement and color shifts in the video. To resolve this, I applied a dark overlay using background-color: rgba(0, 0, 0, 0.7); on top of the video, which improved the contrast and made the text stand out without taking away from the video’s visual appeal.

- Improving Accessibility: Ensuring the page was accessible to users with disabilities was another challenge. I had to ensure that all images had alt text, that there was a clear color contrast between text and backgrounds, and that interactive elements could be easily navigated with a keyboard. I solved these issues by following best practices for semantic HTML and accessibility guidelines, making sure all elements were labeled and easily accessible to screen readers.

Lessons Learned
From overcoming these challenges, I learned the importance of planning the layout and design structure before diving into implementation. I also learned how to be more mindful of accessibility and user experience, ensuring that the page is both functional and inclusive. Additionally, working with video backgrounds and responsive design taught me how to balance aesthetics with usability, ensuring that the final result is visually appealing but also usable on all devices. These challenges were valuable learning experiences that enhanced my understanding of web development, especially in terms of creating interactive and accessible websites.
