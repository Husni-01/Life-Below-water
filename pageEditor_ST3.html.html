Feedback

Technical Discription

I created the feedback page using HTML and CSS. The form allows users to enter their name, email, and provide feedback through radio buttons, a star rating system, and text areas for suggestions. CSS styles the page with a centered form, a shadow effect, and smooth hover interactions. The form layout ensures readability and user-friendliness. Responses are sent via email using the form’s action attribute.

Accessibility

The form improves accessibility by linking labels to inputs for screen readers, using high contrast for readability, and including required fields to guide users. Keyboard navigation is supported for all elements. Hover effects activate on focus for better interaction. Text-based feedback options ensure flexibility, making the form inclusive for all users.

Team

Technical Discription

I created the team page using HTML and CSS to present team members in an interactive and visually appealing way. The page features individual profile cards that showcase each member’s name, role, and contributions. CSS was used extensively to style the layout, add hover effects, and implement a smooth glassmorphic design. Subtle animations, such as fading effects and image scaling on hover, enhance the user experience. The design is fully responsive, ensuring it looks great on different screen sizes.

Accessibility

The team.html page enhances accessibility by using proper heading structures for screen readers, high contrast text for readability, and alt attributes for images. Keyboard navigation is supported for interactive elements. The video background is muted by default to prevent distractions. Clear role separation and structured content improve usability for all users.

Content Page

Technical Discription

The page uses structured HTML elements like header, nav, main,and footer for a well-organized layout. A background video is implemented using the video tag with object-fit: cover to ensure it scales properly. CSS properties like position: fixed, z-index: -1, and background-color: rgba(0,0,0,0.3) create an overlay effect for better readability. Navigation links use display: inline for horizontal alignment. Tips sections are styled with border-left, border-radius, and backdrop-filter: blur(10px) for a modern glass-like effect.

Accessibility

The page enhances accessibility by ensuring proper text contrast against the background. Semantic HTML tags (header, nav, section) improve screen reader compatibility. The alt attributes for images ensure visually impaired users understand content. Keyboard navigation is supported with properly structured nav links. The overlay improves readability by reducing video brightness, making text easier to read.

Challenges and Lessons Learned

During implementation, I faced challenges such as creating a clickable star rating system, setting a video as a background with proper CSS alignment, and implementing a "Go to top" button. To resolve these, I used YouTube tutorials and GitHub Copilot for guidance. This experience taught me the value of utilizing online resources and persistence to overcome obstacles and successfully complete tasks.


Compliance

Accessibility – Proper HTML structure and keyboard navigation are implemented, but some images lack alt attributes, and form labels need improvement.

Data Protection – Minimal data is collected, but mailto: for form submission is insecure. A secure server-side solution is recommended.

Copyright – Generic images are used, but some lack attribution. Verifying copyright and providing credits where necessary is advised.

Security – The site avoids major risks, but the form needs CAPTCHA and validation to prevent spam.

Ethical Use – Content is factual, relevant, and promotes sustainability
